What follows is a dynamically generated list of elements that are currently available in the file system to generate pages. This list is recreated on each build.

If you only see the headings below, make sure you have SSIs enabled on your Apache server (see Introduction to Server Side Includes).

Your first dynamic page is here: Boilerplate masterpage.

Components

These are mainly page layout snippets that may not make much sense when viewed on their own. Components are assembled into pages by templates.

Modules

These are standalone content modules that are included in page view files and rendered by a template.

Pages

Pages are generated by calling a template that includes components, and by page view files that include modules.

Templates

Each page controller defines the template to use, and the view (list of modules) to include in the template. Much like components, templates don't make sense when viewed on their own. They are just empty shells where the components and modules are included dynamically, to create full pages that contain different module configurations. Templates are included here only for completeness. You should view templates in your editor instead.
